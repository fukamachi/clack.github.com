Clack

Revenge of Lisp in Web

Chapter 1: Introduction

Clack is a web application environment for Common Lisp, inspired by Python's WSGI, Ruby's Rack and Perl's Plack. By abstracting HTTP into a simple API, Clack allows your web applications to be portable and be reusable.

If you are already familiar with Web development, you may know why these things so important -- because Web applications are in beta forever.

It means we have to improve applications day by day. Your application will be 3 times larger than you thought. The most are the functions that you did not imagine at the time of development. Therefore your application should be clean and should be modulable.

Besides that, there is another problem. Your application would keep running than you imagined. As a result, your application is getting older little by little. Your web server may be old-fashioned 5 years later. Your web framework may be not maintained in the future. Same as libraries you used. What will you do then, if your application heavily depends on a specific product. Accordingly, we have to keep our web applications to be easy to modify.

Clack was developed to solve these problems. Clack works like glue of a web application. Clack isn't a web application framework. Clack is a collection of modular components. A web application based on Clack looks like a bunch of components which talks each other.

Be relieved, it is easy to learn how to use Clack, at least, than learning other web applications. Though I can't justify it in this short introduction, I leave some informations that you can judge whether we worth trusted for instead.

Clack is ready to use

All the functions of the core of Clack are already implemented in the first official release in June, 2011. Since then, no serious bugs have been found and no changes without backward compatibility have done. Clack seems stable.

Clack is getting better

Clack is released every month. There are many kind of changes in them, but, I really wanna say is, continuous improvement has been done. In the process, a function of Clack has been strengthened steadily, and even if a fatal bug was found in future by any chance, we can expect it is fixed by next month.

Clack is well tested

The reason which only a few bugs were found so far is many unit tests. It has 173 tests now. The test coverage is kept more than 70% (I wanna improve it more, though).

Of course, all releases had passed those all tests, not only on 1 CL implementation but 3 -- Clozure CL, SBCL and CLISP. You can check the current status on http://ci.clacklisp.org/.
